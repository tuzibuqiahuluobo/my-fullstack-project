# my-backend

> **Languages:** English | [简体中文](README.zh-CN.md)

A lightweight Go HTTP backend providing user registration, login, and profile update APIs.

## Features

- User registration with bcrypt password hashing and email verification (QQ & Gmail)
- User login (returns `uid`, `avatar`, `username`, `nickname`)
- Profile updates (username, avatar, password, nickname — each field optional, username updates are rate-limited)
- Post management (create, retrieve all, delete by owner or admin)
- User management (retrieve all users, delete any user by admin)
- Auto-generates default nickname and avatar on registration
- Auto-creates super admin account on first run if not present
- SQLite auto-migration on startup
- CORS enabled for frontend development

## Tech Stack

| Dependency | Purpose |

## Quick Start

- Go 1.26.4 or later

```bash
git clone https://github.com/tuzibuqiahuluobo/my-fullstack-project
```

## Data Model

The `User` struct in `models.go`:

| Field | Type | Description |
|-------|------|-------------|
| `uid` | uint | Primary key, auto-increment |
| `username` | string | Unique username |
| `password_hash` | string | bcrypt hash (not exposed via API) |
| `avatar` | string | Avatar URL, auto-generated by DiceBear if not provided |
| `email` | string | User's email, used for registration and verification |
| `role` | int | User role (0: regular, 2: super admin) |
| `username_updated_at` | time.Time | Timestamp of last username update (rate-limited to 180 days) |
| `nickname` | string | User's display name, auto-generated as `user_{UID}` by default |

The `Post` struct in `models.go`:

| Field | Type | Description |
|-------|------|-------------|
| `id` | uint | Primary key, auto-increment |
| `username` | string | Username of the post author |
| `nickname` | string | Nickname of the post author (cached at creation) |
| `avatar` | string | Avatar URL of the post author (cached at creation) |
| `content` | string | Content of the post |
| `created_at` | time.Time | Timestamp of post creation |

## API Reference

All endpoints use `POST` with `Content-Type: application/json`. CORS is enabled (`Access-Control-Allow-Origin: *`).

### Register

**`POST /api/register`**

Request body:

```json
{
  "username": "alice",
  "password": "secret123",
  "email": "test@qq.com",
  "code": "123456"
}
```

Success response (`200`):

```json
{"message": "注册成功！欢迎加入。"}
```

Error responses:

- `400` — Invalid JSON format, username or email already taken
- `401` — Verification code expired or incorrect
- `500` — Password hashing failed

```bash
curl -X POST http://localhost:8080/api/register \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"secret123","email":"test@qq.com","code":"123456"}'
```

### Login

**`POST /api/login`**

Request body:

```json
{
  "username": "alice",
  "password": "secret123"
}
```

Success response (`200`):

```json
{
  "message": "登录成功！欢迎回来，alice",
  "uid": 1,
  "avatar": "https://api.dicebear.com/7.x/adventurer/svg?seed=user_1",
  "username": "alice",
  "nickname": "user_1"
}
```

Error responses (`401`):

- Username does not exist
- Incorrect password

```bash
curl -X POST http://localhost:8080/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"secret123"}'
```

### Update Profile

**`POST /api/update`**

Request body:

```json
{
  "uid": 1,
  "username": "alice_new",
  "avatar": "https://example.com/avatar.png",
  "password": "newpassword",
  "nickname": "Alice Wonderland",
  "current_password": "secret123" 
}
```

`username`, `avatar`, `password`, `nickname` are optional. To update `username`, `current_password` must be provided, and it's limited to once every 180 days.

Success response (`200`):

```json
{"message": "资料更新成功！核心凭证已同步。"}
```

Error responses:

- `400` — Invalid JSON format, username already taken
- `401` — Security verification failed (incorrect current password)
- `403` — Username update rate-limited (180 days), or insufficient permissions
- `404` — User not found
- `500` — Update failed (database error, password hashing error)

```bash
curl -X POST http://localhost:8080/api/update \
  -H "Content-Type: application/json" \
  -d '{"uid":1,"nickname":"Alice Wonderland","current_password":"secret123"}'
```

Success response (`200`):

### Get All Posts

**`GET /api/posts`**

No request body.

Success response (`200`):

```json
[
  {
    "id": 1,
    "username": "alice",
    "nickname": "Alice Wonderland",
    "avatar": "https://example.com/avatar.png",
    "content": "Hello, world!",
    "created_at": "2023-10-27T10:00:00Z"
  }
]
```

Error responses:

- `500` — Database error

```bash
curl http://localhost:8080/api/posts
```

### Create Post

**`POST /api/create-post`**

Request body:

```json
{
  "username": "alice",
  "nickname": "Alice Wonderland",
  "avatar": "https://example.com/avatar.png",
  "content": "This is a new post."
}
```

Success response (`200`):

```json
{"message": "发布成功！"}
```

Error responses:

- `400` — Invalid JSON format or empty content
- `500` — Database error

```bash
curl -X POST http://localhost:8080/api/create-post \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","nickname":"Alice Wonderland","avatar":"https://example.com/avatar.png","content":"This is a new post."}'
```

### Delete Post

**`POST /api/delete-post`**

Request body:

```json
{
  "post_id": 1,
  "username": "alice"
}
```

Success response (`200`):

```json
{"message": "帖子已永久销毁"}
```

Error responses:

- `400` — Invalid JSON format
- `403` — Unauthorized (only post owner or "超级管理员" can delete)
- `404` — Post not found
- `500` — Database error

```bash
curl -X POST http://localhost:8080/api/delete-post \
  -H "Content-Type: application/json" \
  -d '{"post_id":1,"username":"alice"}'
```

### Get All Users (Admin Only)

**`GET /api/users?admin=超级管理员`**

No request body. Requires `admin=超级管理员` query parameter for authorization.

Success response (`200`):

```json
[
  {
    "uid": 1,
    "username": "alice",
    "avatar": "https://example.com/avatar.png",
    "email": "test@qq.com",
    "role": 0,
    "username_updated_at": "2023-10-27T10:00:00Z",
    "nickname": "Alice Wonderland"
  }
]
```

Error responses:

- `403` — Unauthorized (not "超级管理员")
- `500` — Database error

```bash
curl "http://localhost:8080/api/users?admin=超级管理员"
```

### Delete User (Admin Only)

**`POST /api/delete-user`**

Request body:

```json
{
  "admin_name": "超级管理员",
  "target_uid": 2
}
```

Success response (`200`):

```json
{"message": "该用户已被强制剥夺权限并注销账号"}
```

Error responses:

- `400` — Invalid JSON format
- `403` — Unauthorized (not "超级管理员")
- `500` — Database error

```bash
curl -X POST http://localhost:8080/api/delete-user \
  -H "Content-Type: application/json" \
  -d '{"admin_name":"超级管理员","target_uid":2}'
```

## Development Notes

- `data.db` is listed in `.gitignore` and will not be committed. Clone the repo and run the server to generate it locally.
- No JWT or session mechanism is implemented. The frontend should store `uid`, `username`, `nickname`, and `avatar` after login.
- CORS is set to `*` for easy local development. Restrict this in production.
- A super admin account ("超级管理员") with default password "ASDasd5201314." (can be changed in `models.go`) is automatically created on first run if it doesn't exist. Its role is `2`. You can use this account for admin API testing.
